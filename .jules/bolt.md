## 2024-05-21 - Memory Allocation Optimization for File Operations
**Learning:** When using `sync.Pool` to manage `[]byte` slices in Go to avoid allocations, store and retrieve pointers to the slice (`*[]byte`) instead of the slice directly. Converting a `[]byte` to an `interface{}` when putting it into the pool natively causes an additional heap allocation. Furthermore, when the required buffer size depends on a mutable global variable (like `BufferSize`), always verify the capacity of the retrieved slice, reading the global once into a local variable to prevent race conditions.
**Action:** Always pool pointers to slices instead of the slices themselves, and carefully validate dynamic sizing requirements when retrieving from `sync.Pool`.
