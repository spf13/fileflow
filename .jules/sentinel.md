## 2024-05-24 - TOCTOU Symlink Vulnerability in file creation
**Vulnerability:** The `fileflow.Copy` function used `os.OpenFile` to create the destination file directly. If an attacker created a symlink at the destination path between the `Exists` check and the `OpenFile` call, the program would overwrite the target of the symlink (Time-of-Check to Time-of-Use vulnerability). Additionally, if a failure occurred mid-copy, a partially written file would be left behind.
**Learning:** In Go, simply appending `os.O_EXCL` to `os.OpenFile` breaks overwrite functionality. To prevent TOCTOU symlink vulnerabilities and ensure atomic writes without leaving partial files on failure, the atomic write pattern (temp file + rename) must be used.
**Prevention:** Always use `os.CreateTemp` in the target directory to write data, followed by `destFile.Chmod(mode)` to set permissions, and then `os.Rename` to atomically overwrite the destination. Ensure temporary files are cleaned up in a `defer` block using a pattern like `defer func() { if destFile != nil { destFile.Close(); os.Remove(tmpName) } }()`.
