## 2024-12-02 - Prevent symlink local file overwrite vulnerability
**Vulnerability:** In `fileflow.Copy`, the destination file was opened with `os.O_TRUNC`. Since `Exists(dst)` uses `os.Stat(dst)`, it followed symlinks. If `dst` was a broken symlink, `Exists` returned false. The subsequent `os.OpenFile(..., os.O_CREATE|os.O_TRUNC)` would follow the broken symlink and overwrite or create the target file of the broken symlink, causing a local file overwrite or creation vulnerability.
**Learning:** Checking for file existence and then opening a file introduces a Time-Of-Check to Time-Of-Use (TOCTOU) vulnerability where an attacker can replace the destination file with a symlink. Even without TOCTOU, `os.Stat` follows symlinks, so it misses broken symlinks, allowing them to be followed during `os.OpenFile` if `O_TRUNC` is used.
**Prevention:** Use `os.O_EXCL` in conjunction with `os.O_CREATE` to prevent following any symlinks and failing if any type of file (including broken symlinks) exists at the path, effectively mitigating the symlink attack.
