## 2026-05-17 - TOCTOU Vulnerability in File Copy
**Vulnerability:** The `Copy` function previously used `os.OpenFile(dst, os.O_RDWR|os.O_CREATE|os.O_TRUNC, sourceInfo.Mode())` directly on the destination path. This introduces a Time-Of-Check to Time-Of-Use (TOCTOU) vulnerability where an attacker could replace the destination path with a symlink after the existence check but before the file is opened/truncated, potentially allowing arbitrary file overwrites or unauthorized access. Appending `os.O_EXCL` prevents overwriting existing files, which breaks the intended `Copy` functionality.
**Learning:** File overwriting operations must be robust against TOCTOU symlink attacks without sacrificing the ability to overwrite files when intended.
**Prevention:** Implement an atomic write pattern by creating a temporary file in the same directory using `os.CreateTemp`, writing the content to it, applying necessary permissions, and then atomically renaming it to the final destination using `os.Rename`. Ensure proper resource cleanup with `defer` and prevent double-closing leaks.
