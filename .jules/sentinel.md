## 2024-05-29 - TOCTOU Symlink Vulnerability in File Copy
**Vulnerability:** In `fileflow.go`, `Copy` opens a destination file with `os.O_RDWR|os.O_CREATE|os.O_TRUNC`. However, a malicious actor could create a symlink at the destination path between the time `Exists` is called and `os.OpenFile` is called, potentially causing `os.OpenFile` to overwrite an arbitrary file (if `dst` is in a shared directory, like `/tmp`). The same TOCTOU (Time of Check to Time of Use) issue exists for any file operations that check and then write.
**Learning:** `os.O_EXCL` fails if the file already exists, which breaks standard file copy operations if we want to overwrite. Using a temporary file in the destination directory and then performing an atomic `os.Rename` is the robust cross-platform way to write files atomically and prevent symlink attacks.
**Prevention:** Use the atomic write pattern: `tmpFile := os.CreateTemp(...)`, write to it, `tmpFile.Close()`, `tmpFile.Chmod(...)`, then `os.Rename(tmpFile.Name(), dst)`.
