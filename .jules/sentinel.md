## 2026-05-25 - Prevent TOCTOU symlink vulnerabilities in file operations
**Vulnerability:** The `Copy` function in `fileflow.go` used `os.OpenFile` with a fixed destination path directly. This exposed a Time-of-Check to Time-of-Use (TOCTOU) vulnerability where an attacker could replace the target file with a symlink between the existence check and the `os.OpenFile` call, leading to arbitrary file overwrite. Simply adding `os.O_EXCL` isn't sufficient as the `Copy` function is expected to overwrite in certain conditions (e.g., when the target exists but isn't identical, and we rename).
**Learning:** In Go, standard file overwrites are susceptible to TOCTOU if they follow existence checks or operate in world-writable directories. The secure pattern is an atomic write: creating a temporary file in the destination directory, writing the contents, applying necessary permissions, and then performing an atomic `os.Rename`.
**Prevention:** Always use the atomic write pattern (`os.CreateTemp` + `os.Rename`) for file copy/overwrite operations instead of direct `os.OpenFile` with `os.O_TRUNC`, particularly in utilities that may operate in shared directories.
