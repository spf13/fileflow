## 2025-05-26 - Prevent TOCTOU symlink vulnerability in Copy
**Vulnerability:** The `Copy` function creates or overwrites the destination file using `os.OpenFile(dst, os.O_RDWR|os.O_CREATE|os.O_TRUNC, sourceInfo.Mode())`. A Time-of-Check to Time-of-Use (TOCTOU) symlink vulnerability occurs if an attacker replaces the file with a symlink between the time it is checked/opened and when it's written.
**Learning:** `os.OpenFile` without `os.O_EXCL` allows following symlinks, which could result in an attacker tricking the process into overwriting arbitrary files they wouldn't normally have access to. In this codebase, appending `os.O_EXCL` breaks expected overwrite functionality.
**Prevention:** Use the atomic write pattern. Create a temporary file in the destination directory (`os.CreateTemp(filepath.Dir(dst), ...)` to avoid cross-device rename issues), write the contents, sync, and then perform an atomic `os.Rename()` to safely overwrite the destination. Ensure to `Chmod` the temporary file to securely copy the original permissions.
