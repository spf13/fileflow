## 2026-04-18 - Prevent TOCTOU symlink vulnerabilities in file copy operations
**Vulnerability:** The `Copy` function creates the destination file directly with `os.OpenFile(dst, os.O_RDWR|os.O_CREATE|os.O_TRUNC, sourceInfo.Mode())`. This is susceptible to Time-of-Check to Time-of-Use (TOCTOU) symlink vulnerabilities. An attacker could replace the destination path with a symlink to a sensitive file between the check and the open operation, causing the application to overwrite the sensitive file.
**Learning:**  We cannot simply add `os.O_EXCL` because `Copy` needs to support overwriting files in some cases, and doing so breaks this functionality.
**Prevention:** Use an atomic write pattern. Create a temporary file in the destination directory using `os.CreateTemp(filepath.Dir(dst), "fileflow-*.tmp")` to prevent cross-device rename failures. Write the data to the temporary file, sync, close, and finally use `os.Rename(tmpName, dst)` to atomically replace the destination file.
