## 2024-05-04 - TOCTOU vulnerability in file write operations
**Vulnerability:** The `Copy` function was vulnerable to a Time-of-Check to Time-of-Use (TOCTOU) symlink attack. It used `os.OpenFile` with a known destination path, allowing an attacker to swap the file with a symlink before it was written to, potentially overwriting unintended files with the original file's content or changing permissions.
**Learning:** Using `os.OpenFile` directly with the destination path is risky, especially when the path is constructed or predictable, and doesn't provide atomicity. Also, simply adding `os.O_EXCL` breaks the intended functionality when overwriting files.
**Prevention:** Always use the atomic write pattern: create a temporary file in the destination directory (`os.CreateTemp(filepath.Dir(dst), ...)`), write the data, sync, apply intended permissions securely (`Chmod(sourceInfo.Mode())`), close, and perform an atomic `os.Rename()` over the intended destination. This prevents cross-device rename failures and symlink swapping.
