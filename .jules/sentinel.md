## 2024-05-13 - [Fix TOCTOU vulnerability in file copy]
**Vulnerability:** The `Copy` function creates the destination file directly, leading to a Time-of-Check to Time-of-Use (TOCTOU) vulnerability where a concurrent process could replace the file with a symlink before data is written.
**Learning:** In this codebase, to prevent TOCTOU (Time-of-Check to Time-of-Use) symlink vulnerabilities during file writes/copies, do not simply append `os.O_EXCL` to `os.OpenFile` as it breaks overwrite functionality. Instead, use an atomic write pattern: create a temporary file in the destination directory using `os.CreateTemp(filepath.Dir(dst), ...)` (to prevent cross-device rename failures), write the data, and then perform an atomic `os.Rename()` over the intended destination. Ensure that permissions are applied to the temporary file via `Chmod` before renaming.
**Prevention:** Always write new or replaced file data to a temporary file first, and only move it to its final destination using `os.Rename` once the write operations are complete and successful.
