## 2024-05-01 - Prevent TOCTOU via Atomic Rename
**Vulnerability:** The fileflow package used `os.OpenFile(dst, os.O_RDWR|os.O_CREATE|os.O_TRUNC, mode)` to overwrite a destination file. This is vulnerable to a Time-of-Check to Time-of-Use (TOCTOU) symlink attack. An attacker can replace `dst` with a symlink right before the `os.OpenFile` call, causing the copy operation to overwrite an arbitrary file on the system.
**Learning:** Using `O_EXCL` in `os.OpenFile` prevents following a symlink if the file doesn't exist, but it inherently breaks standard overwrite functionality (where you *want* to overwrite an existing file). To safely overwrite while preventing symlink attacks, a temporary file must be used.
**Prevention:** Always use the atomic write pattern for safe file overwrites: `os.CreateTemp` in the target directory (to avoid cross-device rename failures), explicitly apply the target mode using `os.Chmod`, write the payload, close the file, and finally use `os.Rename(temp, dst)` to atomically overwrite the destination, avoiding symlink race conditions completely.
